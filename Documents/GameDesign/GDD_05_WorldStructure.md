# 第5章：世界结构与关卡设计

## 5.1 箱子的空间结构

整个游戏世界在空间上抽象为一个**区域图（Zone Graph）**，区域之间的连通性由解谜进度决定：

```
              [起始区域 · 核心]
                     │
           ┌─────────┴─────────┐
        [区域A]             [区域B]
        │    │               │
    [区域A'] [区域A'']     [区域C]──[区域D]
   （初始不可达）（初始不可达）    │
                           [区域E]──[出口前室]
                                        │
                                    [出口区域]
```

- **节点（Zone）**：每个区域是相对独立的空间单元，拥有主题性谜题与视觉风格。
- **边（Connection）**：区域间的连接点初始可能封闭（隔离屏障），解谜后开放。
- **重力方向**决定了玩家能否自然"坠入"相邻区域；部分通道只在特定重力角度下可穿越。
- **子区域**（如 A'、A''）：初始不可达，需在其他区域获得新能力后回访解锁。

---

## 5.2 区域隔离机制

区域之间的隔离屏障有以下几种类型，每种对应不同的解除方式：

| 屏障类型 | 描述 | 解除方式 |
|----------|------|----------|
| **重力锁** | 屏障只在特定重力方向下消失 | 将全局重力调整到对应方向后自然穿越 |
| **时间锁** | 屏障处于时间冻结状态，无法物理穿越 | 在该区域完成特定时间谜题，解除冻结 |
| **能量壁垒** | 需要积累足够的 GE 或 CE 才能充能穿透 | 在周边区域积累能量后冲关 |
| **机关锁** | 由区域内的机关链控制开关状态 | 完成该区域的核心谜题序列 |
| **跨区域锁** | 需要同时操控两个区域的机关 | 利用跨区域的物体、重力场或时间线索联动触发 |

---

## 5.3 能力解锁与区域开放的对应关系

玩家在区域中完成核心谜题后获得**能力碎片**，碎片积累到一定数量时解锁新的操控维度，同时开放新区域：

| 能力层级 | 解锁条件 | 新增操控能力 | 开放内容 |
|----------|----------|-------------|----------|
| **基础层** | 初始能力 | 全局重力方向切换（4向）/ 时间回溯（5s） | 起始区域、区域A、区域B |
| **第二层** | 解开2个区域核心谜题 | 重力强度调节 / 慢动作 | 区域C、区域D |
| **第三层** | 解开4个区域核心谜题 | LGF（局部重力场） | 区域E；回访解锁 A'、B' 等子区域 |
| **第四层** | 解开6个区域核心谜题 | LCF（局部时间场） | 出口前室；回访解锁更深层子区域 |
| **最终层** | 集齐所有能力 | 局部时间回溯 / 斜角重力（8向）/ 自由角度 | 出口区域（箱子的逃脱路径） |

> **注**：**引力时间膨胀**为客观物理规律，从游戏开始即持续生效，不属于可解锁能力，故不列入上表。玩家在探索过程中会自然感知并学习利用这条规律。

## 5.4 非线性探索设计原则：区域间双向依存

这是箱子结构最核心的设计哲学。每对存在依存关系的区域遵循以下模式：

```
区域 A（早期进入）                   区域 B（后期进入）
─────────────────                    ─────────────────
主谜题：可部分推进                    主谜题：依赖来自A的道具/线索
存在不可达子区域 A'  ←─ 在B获得新能力后，可回访 A'
  A' 中隐藏：                         B 的谜题解法需要：
    · 道具 X，或                         · 道具 X，或
    · 环境线索 Y，或                     · 线索 Y 的验证，或
    · 可远程激活的机关 Z                 · 机关 Z 的协同触发
```

**设计要求**：

- 每个「不可达子区域」必须对至少一个其他区域的谜题有**实质性帮助**（而非纯粹收集奖励）。
- 玩家回访旧区域时，应有**明确的感知暗示**（如机关发出新的光效或音效），而非无目的漫游。
- 核心谜题必须有**不依赖跨区域资源的基础解法**；跨区域资源提供**更优解或隐藏奖励**，防止因探索顺序不同而卡关。
- 鼓励玩家在同一次重力操控动作中**同时完成移动与解谜**，淡化「移动」与「解谜」的边界。

## 5.5 谜题设计三段式结构

每个区域的核心谜题建议遵循：

```
① 展示（Show）
   → 在安全状态下让玩家观察机关如何响应重力/时间变化

② 约束（Constrain）
   → 限制某种资源（GE不足 / CE不足 / 时间有限），
     迫使玩家寻找联动解法而非单一操控

③ 反转（Invert）
   → 在玩家以为掌握规律后，引入新变量（敌人、额外机关、跨区域依赖），
     要求重新组合已有机制，或回溯旧区域取得关键资源
```

---

## 5.6 重力能量与时间能量的区域设计平衡

- **GE 丰富，CE 匮乏的区域**：鼓励玩家多用重力，通过重力被动影响时间（节省 CE）。
- **CE 丰富，GE 匮乏的区域**：鼓励玩家精妙使用时间，精确控制物体轨迹（弥补重力操控的不足）。
- **双资源匮乏的区域**：极限挑战，要求玩家找到跨区域的能量补给路线或最优操作路径。
- **跨区域能量规划**：玩家可在某区域积累能量（如零重力区域快速回复 CE），带入另一区域使用，形成跨区域的资源流通策略。

## 5.7 终局设计：逃出箱子

逃出箱子并非一扇简单的"通关门"，而是一套需要**组合全部能力**的复合谜题序列：

- **出口的位置**：位于地图的几何中心，被所有其他区域包围。玩家需要从多个不同方向依次"坠入"，最终汇聚于此。
- **出口的激活**：要求玩家在特定时间窗口内同步完成多个子谜题，涵盖慢动作、LGF、LCF 等全部核心技能的综合运用。
- **逃脱的叙事演出**：玩家将全局重力归零，触发终极时间回溯，将「箱子的建造过程」倒放——从而找到并穿过建造之初留下的、现在才得以显现的缝隙，完成逃脱。

## 5.8 休息站（Rest Station）与失效区域

### 5.8.1 在区域图中的位置

休息站是箱子中的特殊节点，出现在区域图的**各大区域群组之间**：

```
[区域A群组] ──── [休息站 R1] ──── [区域B群组]
                      │
               存档 / 升级 / 定制
```

- 玩家完成一个区域群组的核心谜题后，通常可以到达就近的休息站。
- 休息站**不包含谜题**，不受重力/时间机制约束，是纯功能性节点。
- 休息站在区域图中始终**可达**（不受隔离屏障封锁），保证玩家随时能找到存档点。

### 5.8.2 时间回溯失效区域

休息站在技术上是**时间回溯失效区域（Rewind-Free Zone）**：

- 进入时清空历史 Ring Buffer，时间回溯能力禁用（详见 GDD §3.4）。
- 离开后回溯能力恢复，但**无法**回到进入休息站之前的时刻。
- 这使每次进入休息站等同于"确认当前局面"，与存档的叙事语义对齐。

### 5.8.3 休息站提供的功能

| 功能 | 说明 |
|------|------|
| 存档 | 保存当前探索进度与玩家状态 |
| 技能升级 | 消耗探索中获取的道具，提升 GE/CE 上限、LGF 数量、回溯时长等 |
| 玩家定制 | 调整操控映射、查看已积累的谜题线索与地图信息 |
| CE 快速充满 | 进入休息站时 CE 自动回复至满值（奖励机制） |
