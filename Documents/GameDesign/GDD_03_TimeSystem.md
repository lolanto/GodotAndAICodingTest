# 第3章：时间系统

## 3.1 时间回溯

### 3.1.1 基本规则

- 玩家按住**回溯键**，时间以 **2×** 速度反向播放（即倒带）。
- 最大回溯时长：**默认 5 秒**（可通过升级/道具延长至 10 秒）。
- 回溯范围：**全局回溯**（默认），或**局部回溯**（高级技能，仅回溯特定物体）。

### 3.1.2 回溯记录内容

以下状态会被记录并在回溯时还原：

| 记录类型 | 内容 |
|----------|------|
| 物理状态 | 所有动态物体的位置、速度、旋转 |
| 重力状态 | 全局重力方向与强度；所有 LGF 的位置、方向、强度 |
| 敌人状态 | 位置、血量、AI 行为状态 |
| 机关状态 | 开关、门、电梯等的激活状态 |
| 玩家状态 | 位置、速度、**血量** |

### 3.1.3 回溯不影响的内容

- 玩家的**主观记忆**（HUD 上的"已知谜题线索"不消失）
- 已解锁的**区域知识点**（获得的提示永久保留）
- 玩家的**时间能量（CE）** 不还原——CE 是时间回溯本身的资源代价，若随回溯还原则玩家可无限连续回溯，失去资源管理意义
- **音效/BGM** 不倒放（体验设计决策，防止听觉不适）

### 3.1.4 时间回溯的视觉表现

| 状态 | 视觉 | 音效 |
|------|------|------|
| 正常流速 | 正常画面 | 正常 |
| 回溯中 | 画面泛蓝 + 粒子向后流动 + 轻微色差分离 | 低沉的倒带嗡鸣 |
| 回溯结束 | 画面短暂白闪 + 恢复正常色调 | 清脆的"锁定"音 |

---

## 3.2 时间流速控制

除回溯外，玩家可以主动改变**时间流速**：

### 3.2.1 慢动作（Bullet Time）

- 触发后，全局时间流速降至 **0.5×**（`time_scale` 下限）。
- 玩家、敌人、投射物、物理物体全部减速。
- 用途：精确放置 LGF、在高速飞行物之间穿行、精准瞄准。

### 3.2.2 时间加速（Time Rush）

- 触发后，全局时间流速升至 **2×**。
- 玩家角色**同样加速**（操控难度上升）。
- 用途：快速推进耗时机关，在有限时间内完成更多操作。

### 3.2.3 局部时间场（Local Chrono Field，LCF）

高级技能，与 LGF 对应：

- 在特定区域内放置时间气泡，区域内时间流速独立。
- 可与 LGF 共存，形成"局部重力+局部时间"的复合场域谜题。
- 例如：在陷阱区域放置 LCF（时间极慢），在安全区域正常通行。

---

## 3.3 时间资源与限制

时间操控消耗**时间能量（Chrono Energy, CE）**：

> 所有 CE 消耗速率均以**真实时间**计，与当前 `time_scale` 无关。

| 操作 | CE 消耗速率（真实秒） |
|------|------------|
| 时间回溯 | 12 CE/秒（默认 5s 完整回溯消耗 30 CE；升级后 10s 消耗 60 CE） |
| 慢动作（0.5×） | 10 CE/秒 |
| 时间加速（2.0×） | 5 CE/秒 |
| LCF 维持 | 8 CE/秒 · 个 |

> **回溯与时间流速互斥**：触发时间回溯时，慢动作 / 时间加速状态**立即取消**，`time_scale` 强制恢复 1.0，再进入回溯。因此"Time Rush 下回溯 CE 翻倍"的规则不再适用。

- **CE 上限**：默认 100，可升级。
- **CE 回复**：自然回复 5 CE/秒；击败敌人 +20 CE；捡取时间晶石 +30~50 CE。
- **CE 耗尽惩罚**：时间操控强制中断，有 **2 秒冷却**无法重新触发。

> **消耗与恢复互斥**：CE 正在被消耗（回溯 / 慢动作 / 加速 / LCF 维持）期间，自然回复**不会**触发。两者在同一帧内只执行其一。

## 3.4 时间回溯失效区域（Rewind-Free Zone）

### 3.4.1 定义与行为

**失效区域**是箱子中一类特殊区域，玩家进入后时间回溯功能将被**完全禁用**：

| 事件 | 行为 |
|------|------|
| **进入失效区域** | ① 强制中断任何正在进行的回溯；② 清空全部历史帧（Ring Buffer 归零）；③ 时间回溯与慢动作/加速技能均锁定不可触发 |
| **在失效区域内** | CE 正常回复，但所有时间操控技能不可用；历史帧不再积累 |
| **离开失效区域** | 时间回溯能力恢复；Ring Buffer 从当前帧起重新积累（历史为空，**不能**回到进入区域之前的时刻） |

> **关键**：离开失效区域后，玩家将**无法**回溯到进入该区域之前的状态。这是有意的设计约束，与休息站的"档位分隔"功能一致——每次进入休息站相当于确认了当前局面。

### 3.4.2 功能定位：休息站（Rest Station）

失效区域在世界设计中扮演**休息站**角色，提供以下功能：

- **存档**：保存当前游戏进度
- **技能升级**：消耗道具提升能力（GE/CE 上限、LGF 数量、回溯时长等）
- **玩家属性定制**：调整操控偏好、查看已解锁的谜题线索

**设计理由**：上述操作对玩家属性的改动幅度大且多样，若允许时间回溯还原，需要对升级数据、存档状态等复杂系统建立完整的快照机制，实现成本过高。以禁用回溯换取实现可行性，同时赋予休息站"确认当前状态"的叙事意义。

### 3.4.3 视觉与交互提示

| 状态 | 视觉 | HUD |
|------|------|-----|
| 接近失效区域入口 | 门框/边界发出柔和暖光，与箱子内部冷色调形成对比 | 提示"进入区域后时间回溯将暂时失效" |
| 进入失效区域瞬间 | 短暂的暖色闪光 + 粒子向外扩散（象征历史消散） | CE/时间回溯图标变灰并加锁 |
| 在失效区域内 | 环境整体偏暖色调，无蓝色时间残影 | 回溯相关按钮持续显示锁定状态 |
| 离开失效区域 | 短暂的蓝色光晕（象征时间流恢复） | 图标解锁，提示"时间回溯已恢复" |
